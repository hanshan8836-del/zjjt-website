## 🔒 权限与安全说明（重要）

> 本网站虽然**公开可访问**（任何人打开 URL 都能浏览），但**只有仓库所有者（魏小冲）本人可编辑**。

### GitHub 的默认权限模型 = "公开可读 + 私有可写"

- ✅ **任何人都能浏览** `https://<你的用户名>.github.io/zjjt-website/`
- ❌ **没有人能修改代码**——GitHub Pages 只是把你 `main` 分支的静态文件渲染出来
- 🔐 **写入权限完全属于仓库所有者**：访客无法 push、无法 commit、无法改文件
- 🔍 如有 Pull Request 请求，**必须经仓库所有者审批合并**才会生效

### 进一步强化权限的三层方案

1. **基础层**：默认情况下已足够安全（Collaborator 权限默认为空）
2. **审计层**：`Settings → Branches → Add rule` 保护 `main` 分支，禁止 Force Push
3. **隐私层**：如需**完全私有**（访客也不能查看源码），可：
   - 将仓库改为 Private（需 GitHub Pro，$4/月）
   - 或者在 `Settings → Pages → Visibility` 限制（企业账号）
   - 或改用 Cloudflare Pages / Vercel 的访问控制功能

### 推荐操作（免费方案下的最佳安全实践）

```bash
# 1. 启用两步验证（GitHub 账户层面）
#    Settings → Password and authentication → Two-factor authentication

# 2. 保护 main 分支
#    Repo Settings → Branches → Add rule → main
#    ✓ Require pull request reviews before merging
#    ✓ Restrict who can push to matching branches (只允许自己)

# 3. 禁用 Issues / Wiki（减少攻击面）
#    Repo Settings → Features → 取消勾选 Issues / Wiki

# 4. 加一份 CODEOWNERS 文件（此项目已包含）
#    强制所有改动都需你 review
```

## 🌐 部署到 GitHub Pages

### 步骤 1：创建 GitHub 仓库

在 GitHub 上新建一个仓库，取名比如 `zjjt-website`（或者你自己的用户名仓库 `<username>.github.io`）。

### 步骤 2：本地初始化 & 推送

```bash
cd zjjt-website
git init
git add .
git commit -m "🎉 初始化：浙江交工企业门户网站"
git branch -M main
git remote add origin https://github.com/<你的用户名>/zjjt-website.git
git push -u origin main
```

### 步骤 3：启用 GitHub Pages

1. 打开仓库页面 → **Settings** → **Pages**
2. **Source** 选择 `Deploy from a branch`
3. **Branch** 选择 `main` / `(root)`
4. 点击 **Save**
5. 等 1-2 分钟，顶部会出现 `Your site is live at https://<你的用户名>.github.io/zjjt-website/`

### 步骤 4：自定义域名（可选）

在 `Settings → Pages → Custom domain` 填入你的域名，然后在域名服务商处加 CNAME 记录指向 `<你的用户名>.github.io`。
